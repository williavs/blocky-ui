# Card

The Card component is a versatile container for grouping related content and actions. It features our signature blocky style with bold borders and crisp shadows.

## Import

```tsx
import { Card } from '@blocky/react';
```

## Usage

```tsx
<Card>
  <h3>Card Title</h3>
  <p>Card content goes here.</p>
</Card>
```

  <div className="grid grid-cols-2 gap-4">

      <h4 className="font-bold mb-2">Statistics</h4>
      <p className="text-green-900">Monthly overview</p>

      <h4 className="font-bold mb-2">Activity</h4>

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes to apply |
| `children` | `ReactNode` | `undefined` | The content to render inside the card |

## Design Guidelines

### When to use

- **Content Grouping**: Use cards to group related information and actions
- **Visual Hierarchy**: Create clear sections in your layout
- **Interactive Areas**: Define clickable or interactive regions
- **Dashboard Elements**: Display statistics, summaries, or status information

### Best Practices

1. Maintain consistent padding within cards (usually 24px)
2. Use clear visual hierarchy with headings and content
3. Keep card content focused and concise
4. Use appropriate spacing between cards
5. Consider card interactions (hover, focus states)
6. Use background variations sparingly

### Layout Patterns

Common card layouts include:

1. **Header + Content**: Title at the top, content below
2. **Media + Content**: Image or media followed by text
3. **Content + Actions**: Content with buttons at the bottom
4. **Stats Display**: Numbers or metrics with labels
5. **List Items**: Repeating content in a consistent format

## Accessibility

The Card component follows accessibility best practices:

- Uses semantic HTML structure
- Maintains proper heading hierarchy
- Ensures sufficient color contrast
- Supports keyboard navigation for interactive elements
- Includes proper ARIA attributes when needed
