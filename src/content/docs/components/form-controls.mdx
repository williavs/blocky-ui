# Form Controls

Our form controls are designed to be accessible, consistent, and beautiful. Each component follows our signature blocky style while maintaining proper form functionality.

## TextArea

The TextArea component is used for multi-line text input. It supports labels, placeholder text, error states, and more.

### Basic Usage

<Card className="p-6 space-y-4">
  <TextArea
    label="Message"
    placeholder="Type your message here..."
  />
</Card>

```tsx
<TextArea
  label="Message"
  placeholder="Type your message here..."
/>
```

### With Error State

<Card className="p-6 space-y-4">
  <TextArea
    label="Message"
    placeholder="Type your message here..."
    error="Please enter at least 20 characters"
  />
</Card>

```tsx
<TextArea
  label="Message"
  placeholder="Type your message here..."
  error="Please enter at least 20 characters"
/>
```

### Disabled State

<Card className="p-6 space-y-4">
  <TextArea
    label="Message"
    placeholder="This textarea is disabled"
    disabled
  />
</Card>

```tsx
<TextArea
  label="Message"
```

## Accessibility

All form controls follow WAI-ARIA guidelines:

- Proper labeling with `aria-label` or `<label>` elements
- Error states announced with `aria-invalid` and `aria-errormessage`
- Focus management with visible focus indicators
- Keyboard navigation support
- Proper role attributes
- Clear state changes announced to screen readers

## Best Practices

1. **Labels**: Always provide clear, descriptive labels
2. **Error States**: Show clear error messages when validation fails
3. **Grouping**: Use fieldsets to group related controls
4. **Validation**: Validate on blur and form submission
5. **Defaults**: Provide sensible default values when applicable
6. **Spacing**: Maintain consistent spacing between form elements
7. **Required Fields**: Clearly indicate required fields
8. **Help Text**: Provide additional context when needed

## Form Layout

For optimal form layout:

1. Use a single-column layout for better readability
2. Group related fields together
3. Align labels and fields consistently
4. Maintain sufficient spacing between sections
5. Place primary actions at the bottom
6. Use clear section headings
7. Keep forms as short as possible
