# Button

A versatile button component that follows our blocky design system with crisp borders, playful shadows, and smooth hover animations.

## Import

```tsx
import { Button } from '@blocky/react';
```

## Examples

### Variants

Our button component comes with several variants to suit different contexts and actions.

<div className="not-prose flex flex-wrap gap-4 [&>button]:min-w-[120px]">
  <Button>
    Default
  </Button>
  <Button variant="outline">
    Outline
  </Button>
  <Button variant="ghost">
    Ghost
  </Button>
  <Button variant="download">
    Download
  </Button>
</div>

```tsx
<Button>Default</Button>
<Button variant="outline">Outline</Button>
<Button variant="ghost">Ghost</Button>
<Button variant="download">Download</Button>
```

### Sizes

Available in three sizes to accommodate different use cases.

<div className="not-prose flex flex-wrap items-center gap-4">
  <Button size="sm">
    Small
  </Button>
  <Button>
    Default
  </Button>
  <Button size="lg">
    Large
  </Button>
</div>

```tsx
<Button size="sm">Small</Button>
<Button>Default</Button>
<Button size="lg">Large</Button>
```

### With Icons

Buttons can include icons before or after the text using `leftIcon` or `rightIcon` props.

<div className="not-prose flex flex-wrap gap-4">
  <Button leftIcon={<Download className="w-4 h-4" />}>
    Download
  </Button>
  <Button variant="outline" rightIcon={<ArrowRight className="w-4 h-4" />}>
    Next Step
  </Button>
  <Button variant="ghost" leftIcon={<Plus className="w-4 h-4" />}>
    Add Item
  </Button>
  <Button variant="download" rightIcon={<Share className="w-4 h-4" />}>
    Share
  </Button>
</div>

```tsx
<Button leftIcon={<Download className="w-4 h-4" />}>
  Download
</Button>

<Button 
  variant="outline" 
  rightIcon={<ArrowRight className="w-4 h-4" />}
>
  Next Step
</Button>

<Button 
  variant="ghost" 
  leftIcon={<Plus className="w-4 h-4" />}
>
  Add Item
</Button>

<Button 
  variant="download" 
  rightIcon={<Share className="w-4 h-4" />}
>
  Share
</Button>
```

### Loading State

Buttons show a spinner when in loading state and automatically disable interaction.

<div className="not-prose flex flex-wrap gap-4">
  <Button isLoading>
    Loading
  </Button>
  <Button variant="outline" isLoading>
    Processing
  </Button>
  <Button variant="download" isLoading leftIcon={<Download className="w-4 h-4" />}>
    Downloading
  </Button>
</div>

```tsx
<Button isLoading>Loading</Button>
<Button variant="outline" isLoading>Processing</Button>
<Button 
  variant="download" 
  isLoading 
  leftIcon={<Download className="w-4 h-4" />}
>
  Downloading
</Button>
```

### Interactive Examples

Buttons feature smooth hover and active states with shadow animations.

<div className="not-prose flex flex-wrap gap-4">
  <Button className="min-w-[200px]" onClick={() => alert('Clicked!')}>
    Click me!
  </Button>
  <Button 
    variant="outline"
    className="min-w-[200px]"
    leftIcon={<Trash2 className="w-4 h-4" />}
    onClick={() => confirm('Are you sure?')}
  >
    Delete Item
  </Button>
</div>

## Design Details

### Styling

- **Border**: 2px solid black border for strong definition
- **Shadow**: 4px offset black shadow that grows on hover
- **Animation**: Smooth translate and shadow transitions
- **Icons**: 16x16px (w-4 h-4) for consistent sizing
- **Font**: Medium weight for good contrast

### States

- **Hover**: Button lifts up (-2px) and shadow grows
- **Active**: Button returns to original position
- **Loading**: Shows spinner, disables interactions
- **Disabled**: 50% opacity, maintains structure

### Accessibility

- Proper button role and semantics
- Keyboard navigation support
- Loading state properly conveyed
- Sufficient color contrast
- Clear focus indicators
